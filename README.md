# CranChess - 跨平台高自由度桌面棋类引擎
CranChess 并非一个单纯的游戏大厅，而是一个旨在彻底解放桌面棋类创造力的底层引擎与现代化启动器。受到高度可扩展游戏生态的启发，平台将繁琐的视觉渲染层与核心规则判定彻底解耦，为玩家和创作者提供了一个硬核且纯粹的沙盒创造环境。

在这里，没有任何写死在底层的游戏规则。无论是古典的国际象棋、围棋，还是融合了量子叠加或多重宇宙概念的颠覆性变种，都可以通过声明式的配置与安全的沙盒脚本被完美解析并呈现。

## 核心架构与技术基底
### 底层护航与异步并发
引擎核心由 Rust 与 Tauri 共同驱动，提供系统级别的性能表现与内存安全。游戏的主逻辑运行在独立的后台并发线程中，这保证了 UI 渲染的绝对丝滑。

### 隔离沙盒与绝对安全
用户编写的棋类规则代码会在基于 Boa 引擎的严格 JavaScript 沙盒中执行。创作者的恶意代码或死循环不会导致引擎客户端崩溃，平台仅提供有限且安全的本地 API 绑定供脚本调用。

### 现代化启动器与极致掌控
平台内置了极其丰富的参数控制面板。玩家可以自由接管渲染管线的缩放精度、粒子并发限制、补间动画的插值过渡，甚至开启物理碰撞箱视图以满足硬核开发者的调试需求。

### 零硬编码与去中心化生态
所有游戏实例、逻辑脚本和美术资源均被视作独立的 Mod 数据包。引擎本身仅仅是一个规则的观察者与执行者，天然支持热拔插与材质包层叠替换，为构建生生不息的棋类生命树生态奠定基础。

## 环境准备与快速启动
请确保你的操作系统中已正确安装 Node.js (建议 18 及以上版本) 以及 Rust 核心工具链。

将本仓库克隆至本地后，在根目录执行 npm install 完成前端管线依赖的装载。随后执行 npm run tauri dev 即可编译并唤起带有热重载机制的引擎启动器实例。在工作室模块完成重构前，你可以直接在本地文件系统中对游戏包进行开发与调试。

## 创作者指南：如何孕育一款全新的棋类
在 CranChess 中创造一款新游戏，不需要你具备修改底层引擎引擎源码的能力。你只需要在 cran_games 目录下建立一个独立的数据包目录，并按照引擎协议完成配置。

### 第一步：构建游戏清单
每一个游戏都必须包含一个 manifest.json 文件，这是该游戏的基因图谱。你需要在这里声明协议版本号与引擎兼容性限制，避免未来引擎大版本更新破坏你的游戏逻辑。更重要的是，你需要在 environment 节点下定义物理棋盘的尺寸、网格坐标的偏移体系，并在 assets_mapping 节点中将本地的图片纹理映射为引擎可识别的实体类型。

### 第二步：接管生命周期钩子
在游戏包的 logic 目录中创建 rules.js，这是你的规则大脑。引擎会在特定的时机调用你暴露的全局函数。最为核心的钩子是 onMoveAttempt，当玩家在界面上尝试移动选中的实体或在空地落子时，引擎会将目标网格坐标与当前全局状态机的数据快照以 JSON 形式传递入沙盒。你需要在此编写纯粹的逻辑判定代码，来决定这一步是否符合你的游戏规则。

### 第三步：下发状态突变指令
一旦判定走法合法，你的脚本绝不能直接篡改传入的状态对象，而是需要返回一个动作指令数组。引擎目前接受的指令包含 MOVE_ENTITY 用于平移现有实体，MUTATE_STATE 用于无中生有地放置新实体或改变实体属性，以及 DESTROY_ENTITY 用于执行吃子或抹除逻辑。引擎后台在收到这些指令后，会自动进行状态回滚栈的压入，并在前端管线中触发顺滑的插值补偿动画。

### 第四步：调用本地加速接口
为了应对诸如国际象棋中“车”的无视距离移动，或是变种游戏中高频的全盘扫描，引擎在沙盒内挂载了 CranCore 全局对象。你可以直接调用 CranCore.raycast 发射虚拟射线来探测网格阻挡，或者利用 CranCore.getPieceAt 精确读取目标坐标的实体数据。这些密集的查询计算会被下沉至 Rust 底层以原生速度执行，确保复杂判定下的极低延迟。
